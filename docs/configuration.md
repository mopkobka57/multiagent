# Configuration Reference

Complete reference for `multiagent.toml` — the single configuration file for the
Multi-Agent Orchestrator. Generated by `multiagent init`, edited manually.

## Overview

`multiagent.toml` lives in your project root (next to the `multiagent/` directory).
The orchestrator reads it at import time via `project_config.py`. If the file is
missing, the system falls back to auto-detection of an `multiagent_specs/` directory
with hardcoded defaults.

## Sections

### `[project]`

Basic project metadata.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `name` | string | directory name | Project display name |
| `description` | string | `""` | One-line project description |
| `app_dir` | string | `"."` | Path to app source code (relative to project root) |
| `docs_dir` | string | `"docs"` | Path to documentation directory |

```toml
[project]
name = "my-app"
description = "Next.js + Prisma + PostgreSQL SaaS platform"
app_dir = "frontend"
```

### `[data]`

Paths for the agent data directory (backlog, specs, context files).

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `dir` | string | `"multiagent_specs"` | Data directory name |
| `specs_dir` | string | `"{dir}/specs"` | Specs directory |
| `context_files` | list[string] | `[product_context.md, agent_insights.md]` | Files loaded into every Orchestrator prompt |
| `foundational_specs` | list[string] | `[]` | Specs loaded into every agent's context |

```toml
[data]
dir = "multiagent_specs"
specs_dir = "multiagent_specs/specs"
context_files = [
    "multiagent_specs/product_context.md",
    "multiagent_specs/agent_insights.md",
]
foundational_specs = [
    "multiagent_specs/specs/_project-conventions.md",
]
```

#### `[data.spec_types]`

Maps task types to spec subdirectories. The task loader uses these to find
spec files for each task type.

| Key | Default | Description |
|-----|---------|-------------|
| `audit` | `{specs_dir}/audit` | Audit spec directory |
| `feature` | `{specs_dir}/features` | Feature spec directory |
| `tech-debt` | `{specs_dir}/tech_debt` | Tech debt spec directory |
| `refactor` | `{specs_dir}/refactor` | Refactor spec directory |
| `bugfix` | `{specs_dir}/bugfix` | Bugfix spec directory |

```toml
[data.spec_types]
audit = "multiagent_specs/specs/audit"
feature = "multiagent_specs/specs/features"
tech-debt = "multiagent_specs/specs/tech_debt"
refactor = "multiagent_specs/specs/refactor"
bugfix = "multiagent_specs/specs/bugfix"
```

### `[quality_gates]`

Shell commands that validate code quality. The `{app_dir}` placeholder is
replaced with the resolved `app_dir` path at load time.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `fast` | string | — | Quick check (run after each implementation step) |
| `full` | string | — | Full build (run at end of task) |

The pipeline runs `fast` (aliased as `tsc`) after each Implementor step, and
both `fast` + `full` (aliased as `build`) at the end of the task.

```toml
[quality_gates]
fast = "cd {app_dir} && npx tsc --noEmit"
full = "cd {app_dir} && npm run build"
```

**Examples for different stacks:**

```toml
# Python / FastAPI
[quality_gates]
fast = "cd {app_dir} && python -m mypy ."
full = "cd {app_dir} && python -m pytest"

# Rust
[quality_gates]
fast = "cargo check"
full = "cargo build --release && cargo test"

# Go
[quality_gates]
fast = "go vet ./..."
full = "go build ./... && go test ./..."
```

### `[visual_testing]`

Configuration for the dev server and visual regression testing.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `dev_command` | string | `"npm run dev -- -p {port}"` | Command to start dev server. `{port}` is replaced at runtime. |
| `dev_port` | integer | `3000` | Port for the dev server |
| `pages` | list[string] | `["/"]` | URL paths to capture screenshots for |

```toml
[visual_testing]
dev_command = "npm run dev -- -p {port}"
dev_port = 3000
pages = ["/", "/dashboard", "/settings"]
```

### `[git]`

Git branch naming strategy.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `main_branch` | string | `"main"` | Stable branch (never touched by agents) |
| `dev_branch` | string | `"auto-dev"` | Staging branch for automated work |
| `branch_prefix` | string | `"auto/"` | Prefix for feature branches |

Feature branches are created as `{branch_prefix}{task_id}` from `dev_branch`.
For example, task `FE5` gets branch `auto/FE5`.

```toml
[git]
main_branch = "main"
dev_branch = "auto-dev"
branch_prefix = "auto/"
```

### `[models]`

Which Claude models to use for each role.

| Field | Type | Default | Allowed Values | Description |
|-------|------|---------|----------------|-------------|
| `orchestrator` | string | `"opus"` | `"opus"`, `"sonnet"`, `"haiku"` | Model for the Orchestrator agent |
| `subagent` | string | `"sonnet"` | `"opus"`, `"sonnet"`, `"haiku"` | Model for all subagents |

```toml
[models]
orchestrator = "opus"
subagent = "sonnet"
```

### `[budgets]`

Resource limits to prevent runaway costs.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `max_tokens_per_task` | integer | `500000` | Max tokens for the entire Orchestrator session |
| `max_turns_per_subagent` | integer | `30` | Max turns per subagent invocation |
| `max_fix_retries` | integer | `3` | Max retries when quality gate fails |

`max_turns` for the Orchestrator is derived as `max_tokens_per_task / 10_000`.

```toml
[budgets]
max_tokens_per_task = 500000
max_turns_per_subagent = 30
max_fix_retries = 3
```

### `[autonomy]`

Controls how much human interaction is required.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `mode` | string | `"batch"` | `"supervised"`, `"batch"`, or `"autonomous"` |
| `human_checkpoints` | list[string] | `["pr_review"]` | Checkpoints that pause for human input |

Available checkpoints: `task_selection`, `pr_review`.

See [Autonomy Modes](autonomy-modes.md) for detailed behavior of each mode.

```toml
[autonomy]
mode = "batch"
human_checkpoints = ["pr_review"]
```

### `[protected_paths]`

Files and directories that agents must never modify. Violations are automatically
detected via `git diff` and reverted after each agent execution.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `paths` | list[string] | See below | Glob patterns for protected paths |
| `exceptions` | list[string] | `["multiagent/output/"]` | Paths excluded from protection |

**Default protected paths:**
```toml
[protected_paths]
paths = [
    "multiagent/",
    "CLAUDE.md",
    ".claude/",
    ".env*",
    ".gitignore",
    "{data_dir}/backlog.md",
    "*/registry.md",
    "{data_dir}/agent_insights.md",
    "{data_dir}/product_context.md",
    "{data_dir}/specs/_*",
]
exceptions = ["multiagent/output/"]
```

**Pattern rules:**
- `dir/` — matches everything under `dir/`
- `file.md` — exact file match
- `dir/_*` — files starting with `_` in `dir/`
- `.env*` — files starting with `.env`

### `[writable_paths]`

Paths that agents are explicitly allowed to write to. Included in agent prompts
for guidance (not enforced — protected paths are the hard guard).

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `paths` | list[string] | `["{app_dir}/", "docs/"]` | Allowed write paths |

```toml
[writable_paths]
paths = ["frontend/", "docs/"]
```

### `[backlog]`

Backlog parsing configuration.

#### `type_map`

Maps task type strings from backlog.md to internal task types. Useful for
non-English backlogs or custom type names.

```toml
[backlog]
type_map = { feature = "feature", tech-debt = "tech-debt", refactor = "refactor", audit = "audit" }
```

Custom type mapping example (Russian types):
```toml
[backlog]
type_map = { "фича" = "feature", "техдолг" = "tech-debt", "рефактор" = "refactor", "проверка" = "audit", "баг" = "bugfix" }
```

## Config Loading Priority

1. **`multiagent.toml`** — if present in project root, all settings are loaded from it
2. **Auto-detect `multiagent_specs/`** — if no TOML, looks for `multiagent_specs/` next to `multiagent/`
3. **Hardcoded defaults** — fallback values in `config.py`

## Rate Limit Settings

These are hardcoded in `config.py` (not configurable via TOML):

| Constant | Default | Description |
|----------|---------|-------------|
| `RATE_LIMIT_MAX_RETRIES` | `50` | Max retries on rate limit |
| `RATE_LIMIT_BASE_DELAY` | `30` | Base delay in seconds |
| `RATE_LIMIT_MAX_DELAY` | `300` | Max delay (5 minutes) |
| `RATE_LIMIT_BACKOFF_FACTOR` | `2` | Exponential backoff multiplier |
| `SERVER_RATE_LIMIT_DELAY` | `1800` | Server restart delay (30 min) |
| `SERVER_RATE_LIMIT_MAX_RETRIES` | `14` | Server restart max retries |
| `AUDIT_COOLDOWN_DAYS` | `14` | Days between audit re-runs |

## Complete Example

### Next.js + Prisma + Tailwind

```toml
[project]
name = "my-saas"
description = "Next.js 14 + Prisma + PostgreSQL + Tailwind v4"
app_dir = "mip-app"

[data]
dir = "multiagent_specs"
specs_dir = "multiagent_specs/specs"
context_files = [
    "multiagent_specs/product_context.md",
    "multiagent_specs/agent_insights.md",
]
foundational_specs = [
    "multiagent_specs/specs/_project-conventions.md",
    "multiagent_specs/specs/_architecture-blocks.md",
]

[quality_gates]
fast = "cd mip-app && npx tsc --noEmit"
full = "cd mip-app && npm run build"

[visual_testing]
dev_command = "npm run dev -- -p {port}"
dev_port = 3000
pages = ["/", "/dashboard", "/blocks/value-proposition"]

[git]
main_branch = "main"
dev_branch = "auto-dev"
branch_prefix = "auto/"

[models]
orchestrator = "opus"
subagent = "sonnet"

[budgets]
max_tokens_per_task = 500000
max_turns_per_subagent = 30
max_fix_retries = 3

[autonomy]
mode = "batch"
human_checkpoints = ["pr_review"]

[protected_paths]
paths = [
    "multiagent/",
    "CLAUDE.md",
    ".claude/",
    ".env*",
    ".gitignore",
    "multiagent_specs/backlog.md",
    "multiagent_specs/specs/_*",
]
exceptions = ["multiagent/output/"]

[writable_paths]
paths = ["mip-app/", "docs/"]

[backlog]
type_map = { feature = "feature", tech-debt = "tech-debt", refactor = "refactor", audit = "audit" }
```

### Python / FastAPI

```toml
[project]
name = "my-api"
description = "FastAPI + SQLAlchemy + Alembic REST API"
app_dir = "."

[data]
dir = "multiagent_specs"

[quality_gates]
fast = "python -m mypy src/"
full = "python -m pytest && python -m mypy src/"

[visual_testing]
dev_command = "uvicorn main:app --port {port}"
dev_port = 8000
pages = ["/docs"]

[git]
main_branch = "main"
dev_branch = "auto-dev"

[models]
orchestrator = "opus"
subagent = "sonnet"

[protected_paths]
paths = ["multiagent/", ".env*", "alembic/"]
exceptions = ["multiagent/output/"]

[writable_paths]
paths = ["src/", "tests/"]
```
